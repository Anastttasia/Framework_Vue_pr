### Результаты работы 1 и 2 лабараторной работы
 - создана проектная структура и базовая страница со списком задач
 - реализованы UI компоненты для отображения и управления задачами

 Как все работает:

 Интерфейс - это определение кастомного типа данных, но без реализации.

**Функции Vu:**
 - **ref** (из Vue) — реактивная ссылка, которая делает данные отслеживаемыми (при изменении Vue автоматически обновит интерфейс)
 - **computed** (вычисляемое свойство) — это встроенная функция (или опция), которая используется для вычисления и кэширования значений на основе других реактивных данных
 - модификатор **.prevent** отменяет стандартную перезагрузку страницы
 - данные автоматически сохраняются между сессиями благодаря опции **persist: true** (в localStorage)
-  **formRef** — это стандартное имя переменной для template ref (ссылки на HTML-элемент или компонент), которое используется для прямого доступа к форме и управления ею из вашего JavaScript/TypeScript кода
- **Dialog** — компонент Quasar для создания модальных окон
- метод **emit** используется для передачи данных от дочернего компонента к родительскому. Он позволяет дочернему компоненту генерировать (вызывать) пользовательские события, на которые родительский компонент может подписаться и выполнить определенные действия или получить отправленные данные.


**main.ts** - главый файл, который является точкой входа в программу

Его основная задача — инициализировать приложение и «привязать» его к реальному HTML-документу, запустив всю цепочку компонентов, включая главный компонент App.vue

**stores/todoStore.ts** - централизованное хранилище данных для списка дел в приложении и работой с их состоянием

- useTodoStore:
defineStore (из Pinia) — функция для создания хранилища (объединяется в себе состояние (state), логику мутаций/действий (actions) и вычисляемые данные (getters))
- todos - массив объектов типа ITodoItem
-Данные автоматически сохраняются между сессиями благодаря опции persist: true (строка 39) в localStorage

Методы:
- с помощью метода `addTodo` добавление новой задачи в массив todos
- с помощью метода `removeTodo` — удаление задачи по ID
- с помощью метода `toggleTodo` — переключение статуса «выполнено»
- возвращаем объект с состоянием (todos) — можно читать и отслеживать изменения
методами (addTodo, removeTodo, toggleTodo) — можно вызывать для изменения состояния


**IndexPage.vue** - содержит форму для добавления новых задач; отображает список существующих задач; обрабатывает действия пользователя (отметить задачу как выполненную, удалить задачу)
- у компонента TodoList есть свойства и события:
- :items="todoStore.todos" — передаёт массив задач из хранилища Pinia в компонент TodoList как пропс items;
- @toggle="toggle" — подписывается на событие toggle от TodoList. При срабатывании вызовет метод toggle() в текущем компоненте;
- @delete="remove" — аналогично подписывается на событие delete, вызовет метод remove().


Компоненты:
**TodoForm** - компонент отображает форму с двумя полями ввода (название и описание задачи) и кнопкой «Добавить». Он: 
- :rules="[...]" проверяет корректность заполнения формы, lazy-rules — проверка срабатывает только при отправке или вызове validate()
isFormValid — вычисляемое свойство. Возвращает true, если в поле title есть хотя бы один непробельный символ; 
-`onSubmit` 
- отправляет данные в хранилище Pinia - store.addTodo; 
- очищает поля после успешного добавления -  title.value = '', formRef;
- визуально отключает кнопку, если форма невалидна.


Без `trim()` пользователь мог бы ввести одни пробелы в поле «Название задачи» — и форма прошла бы валидацию (ведь строка не пустая). После trim() такая строка станет пустой, что корректно заблокирует добавление.

**TodoItem** - компонент отображает карточку одной задачи
- `События (emit)`:
- метод `toggle` — для переключения статуса выполнения (передаёт id задачи? родительский компонент обработает это событие и обновит состояние в магазине Pinia);
- 'delete' — для удаления задачи (передаёт id)
- метод `remove` создаёт диалоговое окно подтверждения через Dialog.create():
- заголовок, сообщение с названием задачи;
- опция persistent: true — окно нельзя закрыть кликом вне его;
- onOk(() => ...) — если пользователь нажал «Удалить», вызывает событие 'delete' с id задачи.

**TodoList** - компонент для отображения списка задач
- отображает список, только если массив items не пуст;
- События (emit):
- 'toggle' — для переключения статуса выполнения (передаёт id задачи);
- 'delete' — для удаления задачи (передаёт id).
- метод `toggle(id)` принимает id задачи от TodoItem вызывает событие 'toggle' с этим id; родительский компонент получит событие и выполнит нужное действие (например, обновит статус в хринилище Pinia).
- метод `remove(id)` аналогично принимает id от TodoItem - генерирует событие 'delete' с id; родительский компонент обработает удаление.
