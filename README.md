 Как все работает:

 Интерфейс - это определение кастомного типа данных, но без реализации.

**Функции Vue**
 - **ref** (из Vue) — реактивная ссылка, которая делает данные отслеживаемыми (при изменении Vue автоматически обновит интерфейс)
 - **computed** (вычисляемое свойство) — это встроенная функция (или опция), которая используется для вычисления и кэширования значений на основе других реактивных данных
 - модификатор **.prevent** отменяет стандартную перезагрузку страницы
 - данные автоматически сохраняются между сессиями благодаря опции **persist: true** (в localStorage)
-  **formRef** — это стандартное имя переменной для template ref (ссылки на HTML-элемент или компонент), которое используется для прямого доступа к форме и управления ею из вашего JavaScript/TypeScript кода
- **Dialog** — компонент Quasar для создания модальных окон
- метод **emit** используется для передачи данных от дочернего компонента к родительскому. Он позволяет дочернему компоненту генерировать (вызывать) пользовательские события, на которые родительский компонент может подписаться и выполнить определенные действия или получить отправленные данные.
- **watch(storedValue, ...)** — Vue‑функция, которая вызывает колбэк при изменении storedValue


**main.ts** - главый файл, который является точкой входа в программу

Его основная задача — инициализировать приложение и «привязать» его к реальному HTML-документу, запустив всю цепочку компонентов, включая главный компонент App.vue

 Композабл (Composable) во Vue 3 — это функция, созданная с использованием Composition API для инкапсуляции и повторного использования логики с состоянием. Она позволяет вынести повторяющийся код (например, работу с таймером, запросы к API, отслеживание размера экрана) из компонентов в отдельные переиспользуемые модули


 **composables/useLocalStorage.ts** - это Vue‑композабл (composable) useLocalStorage, который создаёт реактивную ссылку, синхронизированную с localStorage
- создать реактивное состояние (через ref), которое автоматически сохраняется в localStorage;
- при загрузке приложения восстановить значение из localStorage, если оно есть;
- отслеживать изменения значения и автоматически сохранять их в localStorage;
- обрабатывать ошибки работы с localStorage (например, при переполнении или отключённом хранилище)

- функция `useLocalStorage`
- <T> — дженерик (обобщённый тип), позволяющий использовать функцию для любых типов данных (string, number, object и т. д.);
- key: string — ключ в localStorage, под которым будут храниться данные;
- initialValue: T — начальное значение, которое будет использовано: при первом запуске (если в localStorage нет данных); если в localStorage есть ошибка или данные повреждены;
- : Ref<T> — тип возвращаемого значения: реактивная ссылка Vue нужного типа.

- функция `loadFromStorage` пытается получить строку по ключу key через localStorage.getItem(key).

Если данные есть (if (item)) -> преобразует строку в объект/значение с помощью JSON.parse(item); приводит тип к T (as T).

При любой ошибке (например, повреждённый JSON, недоступность localStorage) -> перехватывает исключение (catch); выводит ошибку в консоль (console.error); возвращает initialValue.

Если данных нет (item — null), тоже возвращает initialValue.

- функция `storedValue` вызывает loadFromStorage() — пытается загрузить значение из localStorage; передаёт результат в ref<T>(...) — создаёт реактивную ссылку Vue;
явно указывает тип as Ref<T> (для уверенности в типе).

Теперь storedValue — реактивное значение, которое содержит данные из localStorage (если они есть и корректны);
или initialValue (если данных нет или была ошибка).

- функция `saveToStorage` преобразует значение value в строку JSON через JSON.stringify(value).
Сохраняет строку в localStorage по ключу key (localStorage.setItem(...)).

- `watch` отслеживает изменения storedValue и автоматически сохраняет их в localStorage vue‑функция, которая вызывает колбэк при изменении storedValue;

(newValue) => saveToStorage(newValue) — колбэк: передаёт новое значение в saveToStorage;

{ deep: true } — опция для глубокого отслеживания: нужна, если T — объект или массив; гарантирует, что изменения вложенных свойств тоже вызовут сохранение.


**composables/useTodoFilter.ts** - это Vue‑композабл (composable) useTodoFilter, который реализует логику фильтрации списка задач (todo‑list)
- FilterType — тип, определяющий возможные значения фильтра (вероятно, объединение строк: 'all' | 'active' | 'done')
- `interface UseTodoFilterReturn` 
- filter — реактивная ссылка (Ref) на текущий тип фильтра (FilterType);
- filteredTodos — вычисляемое реактивное свойство (ComputedRef) с отфильтрованным массивом задач (ITodoItem[]);
- `setFilter` — функция для установки нового значения фильтра (принимает FilterType, возвращает void). Сама функция - поскольку filter — реактивная ссылка, изменение автоматически запускает пересчет filteredTodos
- `useTodoFilter` — функция принимает один параметр: todos — реактивный массив задач (Ref<ITodoItem[]>); возвращает объект типа UseTodoFilterReturn
- const filter - создаёт реактивную ссылку filter с начальным значением 'all' (показывает все задачи); тип FilterType гарантирует, что значение будет одним из допустимых вариантов
- `filteredTodos` — вычисляет отфильтрованный массив задач на основе текущего значения filter.value
- computed — значение пересчитывается автоматически при изменении filter.value или todos.value;
- реактивность: если изменится исходный массив todos, фильтр пересчитается

- функция `useTodoFilter` возвращает filter — текущий тип фильтра; filteredTodos — отфильтрованный массив задач; setFilter — метод для изменения фильтра


**Добавление анимации при добавлении/удалении задач (Quasar transitions)** - добавила в TodoList тег TransitionGroup - это встроенный компонент во Vue, предназначенный для плавной анимации списков

**useTodoSearch.ts** - это Vue‑композабл (composable) useTodoSearch, который реализует логику поиска по списку задач (todo‑list)
- `interface UseTodoSearchReturn` 
- searchedTodos — реактивная ссылка (Ref) на отфильтрованный массив задач (ITodoItem[]);
- clearSearch — функция без параметров, которая очищает поисковый запрос (возвращает void)
- `useTodoSearch` 
- todos — реактивный массив задач (Ref<ITodoItem[]>);
- searchQuery — реактивная строка поискового запроса (Ref<string>).
- возвращает объект типа UseTodoSearchReturn
- Логика работы: если запрос пустой или состоит только из пробелов — возвращает весь массив todos.value (без фильтрации);
оператор ?. защищает от ошибки, если searchQuery.value равен null/undefined.
- Если запрос есть: приводит его к нижнему регистру и убирает пробелы по краям (toLowerCase().trim()); сохраняет в переменную query.
- Фильтрует массив todos.value: для каждой задачи (todo) берёт поле title; приводит title к нижнему регистру (toLowerCase()); проверяет, содержит ли title подстроку query (includes(query)); оставляет только задачи, где условие истинно.

- Функция возвращает объект с двумя свойствами: searchedTodos — отфильтрованный массив задач (реактивный); clearSearch — метод для очистки поискового запроса
